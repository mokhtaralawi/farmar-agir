# Agri Bridge - نظام إدارة سوق الخضروات والفواكه بالجملة

## نظرة عامة

**Agri Bridge** هو نظام ERP احترافي متكامل لإدارة سوق بيع الخضروات والفواكه بالجملة. تم بناؤه باستخدام Django (Python) وPostgreSQL مع واجهة مستخدم PWA (تطبيق ويب تقدمي) تدعم اللغة العربية بالكامل مع اتجاه RTL.

## المميزات الرئيسية

- **نظام محاسبي مزدوج (Double-Entry Accounting)** كامل
- **إدارة الرعية والمقاوته** مع كشف حساب لكل شريك
- **إدارة الاستلامات** من الرعية مع تحديث المخزون تلقائياً
- **إدارة المبيعات** للباعة مع طباعة فواتير حرارية
- **نظام التحصيل** من المقاوته مع سندات قبض
- **صرف مستحقات الرعية** مع تسويات دورية
- **إدارة الصناديق** (CashBox) مع إغلاق يومي
- **إدارة المخزون** مع تنبيهات الانخفاض
- **إدارة المصروفات** مع تصنيفات
- **إدارة الموظفين والرواتب**
- **تقارير شاملة** (مبيعات، استلامات، مخزون، مالي، أرباح وخسائر)
- **تدقيق كامل** (Audit Trail) لكل العمليات
- **واجهة PWA** تعمل على جميع الأجهزة

## التشغيل

```bash
cd E:\pos\agri_bridge\agri_bridge

# تفعيل البيئة الافتراضية
.\venv\Scripts\activate

# تثبيت المتطلبات
pip install -r requirements.txt

# نسخ ملف البيئة
copy .env.example .env

# تطبيق المايقريشنز
python manage.py migrate

# تجميع الملفات الثابتة للإنتاج
python manage.py collectstatic --noinput

# تشغيل الخادم للتطوير
python manage.py runserver 0.0.0.0:8000

# تشغيل الخادم للإنتاج (Gunicorn)
gunicorn agri_bridge.wsgi --bind 0.0.0.0:8000 --workers 3
```

## النشر على DatabaseMart

1. ارفع الملفات إلى الخادم
2. ثبت المتطلبات عبر `pip install -r requirements.txt`
3. أنشئ ملف `.env` بالإعدادات المناسبة (PostgreSQL)
4. طبق المايقريشنز: `python manage.py migrate`
5. جمع الملفات الثابتة: `python manage.py collectstatic --noinput`
6. شغل Gunicorn عبر Procfile

## النظام المحاسبي المزدوج

كل عملية مالية تنشئ تلقائياً قيداً محاسبياً يتضمن:

1. **الاستلام من الرعوي**: مدين (مخزون) / دائن (الرعية)
2. **البيع للمقوت**: مدين (المقاوته) / دائن (عمولات + مخزون)
3. **التحصيل**: مدين (الصندوق) / دائن (المقاوته)
4. **الصرف للرعوي**: مدين (الرعية) / دائن (الصندوق)
5. **المصروفات**: مدين (المصروفات) / دائن (الصندوق)

## التسويات

نظام التسويات يدعم:
- تسوية أسبوعية مع كل رعوي
- حساب العمولة (5%)
- حساب الخصومات والمصروفات
- إنشاء سند صرف تلقائي بالصافي

## ملاحظات النشر

1. في الإنتاج، غيّر `SECRET_KEY` إلى قيمة عشوائية قوية
2. استبدل SQLite بـ PostgreSQL
3. عيّن `DEBUG = False`
4. عيّن `ALLOWED_HOSTS` بشكل صحيح
5. استخدم خادم WSGI/ASGI مثل Gunicorn + Nginx
